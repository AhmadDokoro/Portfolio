Project overview

Campus Marketplace Platform

A campus-first marketplace where users can list items, browse/search efficiently, and manage listings with role-based access.

Java Spring Boot Spring Data JPA Thymeleaf MySQL Cloudinary
GitHub Live Demo Back to projects

Problem statement

Students often need a safe, simple way to buy/sell items within their campus community. The goal is a marketplace where users can list items, browse/search quickly, and keep data consistent with a clean backend design.

Key features

  • User authentication + role-based access (buyer/seller/admin).
  • Create/update/delete listings with image upload to Cloudinary.
  • Search, filters, and pagination for browsing items.
  • Relational modeling with clear ownership (seller → items) and validations.

System architecture

  • Layered structure keeps controllers thin and business logic in services.
  • Repository layer isolates persistence, making features easier to test and change.
  • Cloudinary integration is handled as a service to keep uploads consistent and secure.
  • Pagination + filtering are handled at the query/service layer for performance.

Database design

The schema models users, roles, and listings with clear ownership and constraints to protect data integrity (e.g., seller → items relationships and validation rules).

Deployment & hosting

  • Docker: containerize the app for consistent builds across environments.
  • Hosting: deploy the container to a platform like Render (or similar).
  • Database: connect to MySQL/TiDB using environment-based configuration.
  • Environment variables: keep secrets out of code (DB URL, credentials, Cloudinary keys).

Project overview

Attendance Management System

A database-backed attendance workflow built with JSP/Servlet and MySQL—designed to reduce manual errors and improve record keeping.

Java JSP Servlets MySQL
GitHub Live Demo Back to projects

Problem statement

Manual attendance is time-consuming and error-prone. This system streamlines class attendance with a simple workflow and a MySQL-backed log that can be reviewed and exported.

Key features

  • Role-based login (lecturer/admin/student).
  • Create sessions and record attendance entries.
  • Attendance logs with filtering and basic reporting.

System architecture

  • MVC-style separation between views (JSP), controllers (Servlets), and persistence logic.
  • Database-backed logs provide a single source of truth for reporting/export.
  • Validation logic prevents common issues (e.g., duplicates and invalid sessions).

Database design

A relational schema stores users, sessions, and attendance entries with constraints to protect integrity and keep reporting simple.

Deployment & hosting

  • Package the app and configure DB connection using environment variables.
  • Deploy to a hosting provider or university server environment.
  • Use Docker optionally for consistent runtime configuration across machines.
